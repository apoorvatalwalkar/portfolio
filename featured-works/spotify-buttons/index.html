CONTEXT

About the Project

For this project, I aimed to redesign the dropdown component for Spotify as a case study into creating more accessible and usable components. I chose the Sort By and View As combined dropdown because I think that the current UI is confusing.

Final Design

PART ONE

Inputs and Outputs

First, I looked into how the existing Spotify dropdown for Sort By and View As work currently. I looked through the how the component responded through mouse, keyboard, and touch user input. I think that the dropdown for mouse and keyboard input is confusing. Since the User Interface for the two toggles--Sort by and View as--are not the same.

PART THREE

Redesign Sketches

My plan for the redesign was to split the current button into two separate components. I think that the "Sort by" and "View as" buttons should be separate toggles. The current combined dropdown is unnecessarily confusing. I addressed the main issue I had with the original dropdown: the dropdown elements for the two buttons are not consistent.

The Sketch

FINAL

The Result

The resulting component splits the existing dropdown into two separate dropdowns. The dropdown components are for both buttons are made consistent.

Takeaways

I enjoyed this deep dive into accessibility. I never needed to consider the accessibility of a page in terms of mouse vs. keyboard vs. touch input. Additionally, I liked getting to redesign a website/app that I use multiple times a day.
